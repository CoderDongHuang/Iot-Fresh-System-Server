# 物联网生鲜品储运系统 - 系统架构说明

## 项目概述

本项目是一个基于Spring Boot的物联网管理系统，专注于生鲜品储运过程中的设备监控和数据管理。系统采用微服务架构思想，提供了完整的设备管理、实时数据监控、报警管理和仪表盘统计功能。

## 技术栈

- **后端框架**: Spring Boot 3.2.0
- **数据库**: MySQL 8.0
- **数据访问**: Spring Data JPA
- **安全框架**: Spring Security + JWT
- **消息队列**: MQTT (用于设备通信)
- **缓存**: Redis
- **构建工具**: Maven
- **编程语言**: Java 17

## 核心功能模块

### 1. 设备管理模块

#### 功能描述
- 设备注册、查询、更新、删除
- 设备状态监控（在线、离线、故障、维护）
- 设备信息管理（制造商、型号、固件版本等）
- 设备实时数据获取
- 设备历史数据查询

#### 主要组件
- **DeviceController**: 处理设备相关的HTTP请求
- **DeviceService/DeviceServiceImpl**: 设备业务逻辑处理
- **DeviceManagementService/DeviceManagementServiceImpl**: 设备管理业务逻辑
- **DeviceRepository**: 设备数据访问
- **Device Entity**: 设备实体

### 2. 数据监控模块

#### 功能描述
- 实时数据监控（温度、湿度、光照等）
- 历史数据查询和分页
- 数据统计分析
- 数据可视化支持

#### 主要组件
- **DeviceDataController**: 处理设备数据相关的HTTP请求
- **RealTimeMonitorController**: 处理实时监控请求
- **DataService/DataServiceImpl**: 数据业务逻辑处理
- **DeviceDataRepository**: 设备数据访问
- **DeviceData Entity**: 设备数据实体

### 3. 报警管理模块

#### 功能描述
- 实时报警监控
- 报警记录管理
- 报警处理（解决、关闭、清除）
- 报警统计分析

#### 主要组件
- **AlarmController**: 处理报警相关的HTTP请求
- **AlarmService/AlarmServiceImpl**: 报警业务逻辑处理

### 4. 仪表盘模块

#### 功能描述
- 顶部统计数据展示（设备数量、数据量、报警数量等）

- 数据增长趋势分析
- 系统状态监控

- **DashboardController**: 处理仪表盘统计请求

## 安全架构

### JWT身份验证流程
1. 用户通过用户名/密码登录
2. 服务器验证凭据并生成JWT令牌
3. 客户端在后续请求中携带JWT令牌
4. 服务器通过JwtAuthenticationFilter验证令牌有效性
5. 验证通过后处理业务逻辑

### 权限控制
- 使用Spring Security进行访问控制
- 配置端点级别的权限
- 支持角色基础的访问控制(RBAC)

## 数据流

### 设备数据采集流程
```
IoT Devices → MQTT Broker → MqttController → DeviceDataRepository → Database
```

### 用户请求处理流程
```
HTTP Request → Security Filter → Controller → Service → Repository → Database
                                              ↓
                                    Response ← DTO ← JSON
```

### 实时监控流程
```
WebSocket Connection → RealTimeMonitorController → DeviceDataRepository → Client
```

## API设计原则

### 统一响应格式
```json
{
  "code": 200,
  "msg": "success",
  "data": {},
  "success": true
}
```

### RESTful API设计
- 使用名词而非动词
- 使用HTTP方法表示操作类型
- 使用复数形式的资源名称
- 使用嵌套结构表示资源关系

### URL路径规范
- `/api/device/` - 设备相关资源
- `/api/dashboard/` - 仪表盘相关资源
- `/api/alarm/` - 报警相关资源
- `/api/user/` - 用户相关资源
- `/api/system/` - 系统设置相关资源
