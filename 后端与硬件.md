# 物联网生鲜品储运系统 - 后端与硬件通信设计

## 一、系统概述

本系统采用MQTT协议作为硬件设备与后端系统之间的主要通信方式，实现设备数据的实时采集、状态监控和远程控制。系统支持多种设备类型，包括温度传感器、湿度传感器、光照传感器等，能够满足生鲜品储运环境的监控需求。

## 二、MQTT主题设计（后端↔硬件）

### 1. 数据上报主题（硬件→后端）

基于当前代码实现，系统支持以下MQTT主题：

```
# 实时数据上报（主要主题）
device/{vid}/data
payload: {
  "Tin": 22.5,           // 内部温度（摄氏度）
  "Tout": 18.0,          // 外部温度（摄氏度）
  "Hin": 65,             // 内部湿度（百分比）
  "Hout": 70,            // 外部湿度（百分比）
  "LXin": 300,           // 内部光照强度（勒克斯）
  "LXout": 250,          // 外部光照强度（勒克斯）
  "brightness": 80,      // 亮度值
  "VStatus": 0,          // 设备运行状态（0-停止，1-运行）
  "timestamp": 1620000000 // 数据时间戳（Unix秒）
}

# 报警上报
device/{vid}/alarm
payload: {
  "type": "temperature",  // 报警类型
  "code": 1001,          // 报警代码
  "message": "温度过高",   // 报警信息
  "level": "high"        // 报警级别
}

# 设备状态上报

payload: {
  "VStatus": 0,          // 设备运行状态
  "battery": 85,         // 电池电量（百分比）
  "timestamp": 1620000000 // 状态时间戳
}

# RFID读取上报
device/{vid}/rfid
payload: {
  "pid": "PID001",       // 产品ID
  "time": 1620000000     // 读取时间
}
```

## 三、MQTT消息处理机制

### 1. 消息处理器（MqttMessageHandler）

系统使用 `MqttMessageHandler` 类处理所有MQTT消息：

```java
@ServiceActivator(inputChannel = "mqttInputChannel")
public void handleMqttMessage(@Payload String payload, @Headers MessageHeaders headers) {
    String topic = (String) headers.get("mqtt_receivedTopic");
    
    if (topic != null) {
        String[] topicParts = topic.split("/");
        if (topicParts.length >= 3) {
            String vid = topicParts[1]; // 设备ID
            String messageType = topicParts[2]; // 消息类型
            
            switch (messageType) {
                case "data":
                    // 处理设备数据
                    dataService.processDeviceDataFromMqtt(vid, payload);
                    break;
                case "alarm":
                    // 处理报警数据
                    processAlarmData(vid, payload);
                    break;
                case "status":
                    // 处理状态数据
                    updateDeviceStatusDirectly(vid, payload);
                    break;
                case "rfid":
                    // 处理RFID数据
                    dataService.processDeviceDataFromMqtt(vid, payload);
                    break;
            }
        }
    }
}
```

### 2. 数据解析逻辑

系统支持JSON格式的数据解析：

```java
private DeviceDataDto parseDeviceDataFromJson(String vid, String payload) {
    try {
        DeviceDataDto dto = new DeviceDataDto();
        dto.setVid(vid);
        
        // 解析JSON数据
        Map<String, Object> jsonMap = objectMapper.readValue(payload, Map.class);
        
        // 解析温度数据
        if (jsonMap.containsKey("Tin")) {
            dto.setTin(convertToDouble(jsonMap.get("Tin")));
        }
        if (jsonMap.containsKey("Tout")) {
            dto.setTout(convertToDouble(jsonMap.get("Tout")));
        }
        
        // 解析湿度数据
        if (jsonMap.containsKey("Hin")) {
            dto.setHin(convertToInteger(jsonMap.get("Hin")));
        }
        if (jsonMap.containsKey("Hout")) {
            dto.setHout(convertToInteger(jsonMap.get("Hout")));
        }
        
        // 解析光照数据
        if (jsonMap.containsKey("LXin")) {
            dto.setLxin(convertToInteger(jsonMap.get("LXin")));
        }
        if (jsonMap.containsKey("LXout")) {
            dto.setLxout(convertToInteger(jsonMap.get("LXout")));
        }
        
        // 解析亮度数据
        if (jsonMap.containsKey("brightness")) {
            dto.setBrightness(convertToInteger(jsonMap.get("brightness")));
        }
        
        return dto;
    } catch (Exception e) {
        return null;
    }
}
```

### 3. 设备状态更新

系统自动更新设备状态：

```java
private void updateDeviceStatusDirectly(String vid, String payload) {
    try {
        // 解析状态更新消息
        Integer status = parseStatusFromPayload(payload);
        if (status != null) {
            // 1. 更新设备状态到devices表
            dataService.updateDeviceStatus(vid, status);
            
            // 2. 更新设备数据表中的状态信息
            dataService.updateDeviceDataStatus(vid, status);
        }
    } catch (Exception e) {
        // 错误处理
    }
}
```

## 四、MQTT配置

### 1. 配置文件（application.yml）

```yaml
mqtt:
  broker-url: tcp://localhost:1883
  username: mqtt_user
  password: mqtt_password
  client-id: iot-fresh-server
  qos: 1
  completion-timeout: 5000
```

### 2. MQTT配置类（MqttConfig）

```java
@Configuration
@EnableIntegration
@IntegrationComponentScan
public class MqttConfig {
    
    @Bean
    public MqttPahoClientFactory mqttClientFactory() {
        DefaultMqttPahoClientFactory factory = new DefaultMqttPahoClientFactory();
        MqttConnectOptions options = new MqttConnectOptions();
        options.setServerURIs(new String[]{"tcp://localhost:1883"});
        options.setUserName("mqtt_user");
        options.setPassword("mqtt_password".toCharArray());
        factory.setConnectionOptions(options);
        return factory;
    }
    
    @Bean
    public MessageProducerSupport mqttInbound() {
        MqttPahoMessageDrivenChannelAdapter adapter = 
            new MqttPahoMessageDrivenChannelAdapter(
                clientId + "-inbound", 
                mqttClientFactory(), 
                "device/+/data", 
                "device/+/alarm", 
                "device/+/status", 
                "device/+/rfid"
            );
        adapter.setQos(1);
        adapter.setOutputChannel(mqttInputChannel());
        return adapter;
    }
}
```

## 五、接口规格说明

### 1. 数据格式

- **编码格式**: UTF-8
- **数据格式**: JSON
- **时间戳**: Unix时间戳（秒）
- **温度单位**: 摄氏度
- **湿度单位**: 百分比
- **光照单位**: 勒克斯（lux）

### 2. 通信频率

- **硬件数据上报**: ≤1秒（满足<2秒要求）
- **服务器数据推送**: ≤5秒（满足<10秒要求）
- **心跳包**: 30秒一次
- **报警即时推送**: 实时处理

### 3. QoS等级

```
MQTT QoS设置：
- 报警数据：QoS 2（确保送达）
- 控制指令：QoS 1（至少一次）
- 普通数据：QoS 0（最多一次）
```

### 4. 错误处理

```
HTTP状态码：
- 200: 成功
- 400: 请求参数错误
- 404: 设备不存在
- 500: 服务器错误

MQTT错误处理：
- 重连机制：指数退避
- 离线消息：保留最后一条
- 会话持久化：启用
```
